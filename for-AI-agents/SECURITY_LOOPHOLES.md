# SECURITY_LOOPHOLES.md — Security Risk Register

This is a structured security risk register.

Each entry documents a vulnerability or risk. Once fixed, the entry is **marked Resolved** rather than deleted. This preserves institutional knowledge and prevents the same class of issue from reappearing.

## Entry Template

```markdown
## [ID-001] Title

- **Description:** Brief description of the vulnerability.
- **Affected Components:** Files, routes, services affected.
- **OWASP Category:** e.g. A01 Broken Access Control
- **Severity:** Critical / High / Medium / Low
- **Likelihood:** Certain / Likely / Possible / Unlikely
- **Business Impact:** What happens if this is exploited?
- **Mitigation:** How to fix or mitigate.
- **Status:** Open / In Progress / Resolved
- **Owner:** Who is responsible.
- **Date Added:** YYYY-MM-DD
- **Date Resolved:** YYYY-MM-DD (leave blank if unresolved)
- **Verification Method:** How resolution was confirmed (e.g. test, audit, manual check)
```
